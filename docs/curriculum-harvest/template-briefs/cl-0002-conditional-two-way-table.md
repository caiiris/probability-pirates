### TEMPLATE-BRIEF — conditional-two-way-table

- **Build target:** `src/features/practice/templates/conditional-two-way-table.ts`
- **Topic:** conditional
- **Skills:** conditional-probability, two-way-table, denominator-selection, joint-probability, marginal-probability
- **Retrieval form:** procedural
- **Difficulty range:** 950-1350
- **Learner goal:** Choose the denominator from the condition, not from the whole table or the visually nearby total.
- **Params:** `tableShape: "2x2" | "2x3"`; `rowLabels: [string, string]` from Pascal-authored neutral category pairs; `colLabels: [string, string] | [string, string, string]` from Pascal-authored neutral category sets; `counts: positive integers[][]` with each cell `2..48`, each row total `8..96`, each column total `8..96`, grand total `30..180`; `questionKind: "joint" | "marginal-row" | "marginal-col" | "conditional-row-given-col" | "conditional-col-given-row"`; `targetRow: valid row index`; `targetCol: valid column index`; generated tables must avoid equal row and column totals for the selected target cell when `questionKind` is conditional, so the denominator misconception is visible.
- **Render shape:** Show a completed two-way count table with row totals, column totals, and a grand total. Ask one probability question in Pascal's voice, using "among", "given", or "randomly chosen from this table" to cue the sample space. For conditional items, visually prompt the learner to identify the restricted row or column before entering the final fraction.
- **Solve:** Compute all answers from integer counts and reduce with exact rational arithmetic. For `joint`, return `counts[targetRow][targetCol] / grandTotal`. For `marginal-row`, return `rowTotal[targetRow] / grandTotal`. For `marginal-col`, return `colTotal[targetCol] / grandTotal`. For `conditional-row-given-col`, return `counts[targetRow][targetCol] / colTotal[targetCol]`. For `conditional-col-given-row`, return `counts[targetRow][targetCol] / rowTotal[targetRow]`.
- **Simulate / exact enumeration:** Exact enumeration is primary: expand the table into `grandTotal` labeled synthetic records with row and column attributes, filter by the condition when present, and assert the favorable count and denominator match `solve()`. Optional Monte Carlo vetting samples one record uniformly from the expanded records for joint/marginal questions, or samples uniformly from the restricted condition group for conditional questions; the observed rate should agree with `solve()` under the standard template tolerance.
- **Distractors:** Use the selected cell over `grandTotal` for a conditional question; use the selected cell over the opposite margin (`rowTotal` instead of `colTotal`, or vice versa); swap the target and condition so `P(row | col)` becomes `P(col | row)`; use a row or column total as the numerator; answer the matching marginal probability instead of the conditional probability.
- **Interaction fit:** Best fit is table-cell or row/column highlighting followed by `fill-fraction`. MVP fallback is `fill-fraction` with authored misconception feedback keyed to denominator choice. A strong v2 interaction asks the learner to tap the denominator group first, then the favorable cell.
- **Worked solution outline:** Read the question and name the condition group; highlight the row or column selected by the condition; use that group's total as the denominator; find the cell that also satisfies the requested target; place that cell count over the condition total; reduce the fraction; contrast briefly with the grand-total answer when the learner chose the common denominator distractor.
